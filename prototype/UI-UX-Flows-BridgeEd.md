# BridgeEd：UI/UX 交互与流程说明（Hackathon）

本文档用于在出图前对齐 **信息架构、页面层级、关键交互与端到端流程**。不产出视觉稿，仅描述交互与流程。

---

## 0. 设计原则（与 PRD 对齐）

- **行动优先**：家长看到的内容默认是「今晚能做什么」，其次才是背景说明。
- **老师减负**：发布与反馈尽量「模板化 + 聚合」，避免长对话流。
- **AI 可控**：所有 AI 输出在老师端可预览/可编辑后再发；家长端可追问但需有明确边界（例如「解释概念」而非开放式聊天）。
- **三端闭环**：老师发布 → 家长理解与执行 → 学生完成与反馈 → 老师看板聚合。

---

## 1. 信息架构（三端）

### 1.1 老师端（Teacher）

- **首页 / 仪表盘**：今日待办、最近发布、班级反馈概览
- **发布学习卡**：新建 → 生成预览 → 编辑 → 选择受众 → 发送
- **学习卡详情**：内容、送达状态、阅读/完成、困难点聚合
- **班级/家长列表**（可简化）：仅用于选择推送对象
- **设置**（可选）：班级名、默认语言、签名

### 1.2 家长端（Parent）

- **收件箱**：按时间排序的学习卡列表（未读优先）
- **学习卡详情**：家长版摘要 + 今晚行动 + AI 辅助区
- **学生练习**：从学习卡进入「生成练习」→ 给孩子做 → 查看结果
- **反馈提交**：结构化表单（完成度、困难点、是否需要老师）
- **设置**（可选）：首选语言、通知偏好

### 1.3 学生端（Student）

- **今日任务**：来自家长/老师发起的练习（Hackathon 可简化为「家长发起」）
- **做题页**：题目 + 作答 + 提交
- **感受打卡**：情绪 + 一句话（可选）

---

## 2. 端到端主流程（E2E）

### 2.1 主链路（Happy Path）

1. **老师**在「发布学习卡」输入课堂要点/知识点 → 系统调用 CurricuLLM 生成「家长版摘要 + 3 条行动建议 + 可选语言」→ 老师预览并轻量编辑 → 选择受众（全班/某位家长）→ 发送。
2. **家长**在收件箱收到卡片 → 打开详情 → 阅读「行动区」。
3. 若不理解：**家长**点击「我没看懂」→ 选择解释层级（更短 / 举例）或切换语言 → 获得解释（仍可在同页继续追问 1 次，避免无限聊天）。
4. **家长**点击「给孩子练习」→ 生成 2–3 题 → 进入学生端或展示二维码/链接给孩子（Hackathon 任选一种）→ **学生**提交答案与感受。
5. **家长**提交结构化反馈（是否完成、困难点、是否需要老师）。
6. **老师**在「学习卡详情 / 看板」看到：完成率、困难点词云或列表、学生情绪分布 → 可选「二次说明」再发给全班或部分家长。

### 2.2 实例：今天课堂「一元二次方程」（贯穿三端）

以下用**同一节课**串起三端，便于评审与拍演示视频时照着走；步骤与 **2.1** 一一对应，仅把抽象动作换成具体情境。

**课堂背景（老师输入时可写进「补充说明」）**  
- 主题：一元二次方程（例如 Year 9–10 数学常见进度，具体年级可按你们 demo 设定）。  
- 今天课上重点：**标准形式** ax²+bx+c=0、**因式分解法**求根、（若课上讲了）**求根公式**或**判别式 Δ** 的直观含义。

---

1. **老师端：发布学习卡**（对应 2.1 第 1 步）  
   - 老师在「发布学习卡」里填写标题（如「本周：一元二次方程—求根思路」）、学科/年级、并把课上要点写成一段话（例如：今天练了把方程写成标准形式、用因式分解写成 (x−根1)(x−根2)=0 这种形式）。  
   - 点击「生成家长版」→ CurricuLLM 产出例如：  
     - **家长摘要**：用非术语话说明「一元二次方程是什么」「今天课上练的是哪一步」。  
     - **今晚 3 条行动**（示例，实际以 AI 生成为准）：  
       1. 让孩子口头解释：什么叫「方程的根」、和「因式分解」有什么关系。  
       2. 一起把一道错题改写成 ax²+bx+c=0 的标准形式（家长不用会解，只要会陪孩子对齐项）。  
       3. 用 5 分钟回顾课本例题一步骤，孩子指出「哪一步最容易跳步」。  
   - 老师**预览**，可改掉口吻或删掉某条行动 → 选择受众（全班或某位家长）→ **发送**。

2. **家长端：收件箱 → 打开学习卡**（对应 2.1 第 2 步）  
   - 家长收到一条新卡片，列表摘要可显示第一行动句（例如「今晚先聊清楚：根是什么意思」）。  
   - 进入详情后**先看「今晚就做」三块**，再展开「孩子本周在学什么」。

3. **家长端：可选 —「我没看懂」**（对应 2.1 第 3 步）  
   - 例如家长不熟悉「判别式」「配方」，点击「我没看懂」→ 选「更短解释」或「举例」→ AI 用生活化类比解释（不要求家长掌握全部推导）。  
   - 若家长需要英文或其他语言，切换语言后再看同一块内容。  
   - 同页**追问 1 次**即可，第三次引导去「反馈给老师」。

4. **家长端 → 学生端：出题与作答**（对应 2.1 第 4 步）  
   - 家长点「给孩子 2–3 道小题」→ 系统基于**本课知识点**生成题（例如：整理成标准形式、简单因式分解、判断有几个实根等，难度与课堂一致）。  
   - 通过学生端入口（链接或同设备切换角色）→ **学生**做题并提交。  
   - **学生**再做**感受打卡**：例如选「有点难」+ 一句「因式分解不知道怎么拆」。

5. **家长端：结构化反馈**（对应 2.1 第 5 步）  
   - 家长勾选：今晚任务**完成度**（完成 / 部分 / 未完成）、**最大困难点**（如「因式分解」「符号容易错」）、**是否需要老师单独联系**（是/否）。  
   - 提交后卡片显示「已反馈」。

6. **老师端：看板与可选二次说明**（对应 2.1 第 6 步）  
   - 老师在「学习卡详情 / 看板」看到：该卡**完成率**、家长填的 **Top 困难点**（如多人选「因式分解」）、**学生情绪/一句话**分布。  
   - 老师可点「二次说明」：针对「因式分解」生成一段**更短的课堂回顾**或**一个家庭小游戏式练习**，再推给全班或只推给勾选「需要联系」的家长。

**实例收束**：家长从「知道今天学了方程」→「知道今晚具体陪孩子做什么」→「不懂时能问 AI」→「孩子做题并表达卡点」→「老师看见聚合反馈并补一刀」，与 **2.1** 的闭环一致。

### 2.3 异常与分支（需在交互上预留）

- **老师未编辑直接发送**：允许，但 UI 上突出「建议快速扫一眼」。
- **AI 生成失败**：展示错误 + 「重试」+「改用模板填空」。
- **家长未登录**：收件箱仅展示最近一条（演示用）或引导登录（若实现）。
- **学生端无账号**：由家长在同一设备代填（演示降级方案）或仅展示链接。

## 3. 老师端：页面级流程与交互

### 3.1 登录 → 仪表盘

- **入口**：登录后默认进入仪表盘。
- **仪表盘内容优先级**：
  1. 「新建学习卡」主按钮（最显眼）
  2. 最近一条学习卡的反馈摘要（完成率、Top 困难点）
  3. 待跟进事项（例如：完成率低于 X% 的卡片）

### 3.2 新建学习卡（核心）

**步骤**（建议单页分步或顶部步骤条）：

1. **输入**：课堂标题、知识点、年级/学科（若有）、补充说明（可选）。
2. **生成**：点击「生成家长版」→ 加载态 → 展示三块内容：
   - 家长摘要（短）
   - 今晚 3 条行动（可勾选保留/删除）
   - 语言版本切换（中/英等）
3. **编辑**：老师可改任何一段文字；改完后可「重新生成」局部（可选，Hackathon 可不做）。
4. **受众**：全班 / 指定家长（多选）。
5. **发送**：确认弹窗（展示推送人数）→ 发送成功 → 跳转至该学习卡详情。

**关键交互**：

- 生成结果区 **固定结构**，避免每次长文不同导致家长端困惑。
- 「发送」前必须有 **预览**，符合「AI 可控」。

### 3.3 学习卡详情 / 看板

**老师在此完成「闭环阅读」**：

- 顶部：标题、发送时间、受众范围。
- 中部：老师最终发布版本（与家长看到一致）。
- 底部：**反馈区**
  - 完成率（家长点击「已执行」或提交反馈后）
  - 困难点 Top 3（来自家长结构化字段）
  - 学生情绪分布（若学生端有数据）
- **动作**：「二次说明」→ 预填困难点上下文 → 生成短通知 → 发送。

---

## 4. 家长端：页面级流程与交互

### 4.1 收件箱

- 列表项展示：**标题、发送日期、未读标记、一句行动摘要**（从「行动区」抽第一行）。
- 点击进入详情。

### 4.2 学习卡详情（家长阅读体验）

**布局建议（从上到下）**：

1. **今晚就做**（3 条，可勾选「已完成」逐条勾选）
2. **孩子本周在学什么**（短摘要，可折叠）
3. **需要帮忙时**（AI 区）

**AI 区交互**：

- 主按钮：「我没看懂」→ 展开：选择「更短解释 / 举例 / 翻译」。
- 限制：同页最多 **2 次追问**（防聊天化）；第三次引导「把困难点提交给老师」。

### 4.3 生成练习 → 学生

- 按钮：「给孩子 2–3 道小题」→ 生成 → 展示题目列表。
- **进入方式**（二选一即可）：
  - **方案 A**：生成「学生链接」家长在手机上打开给孩子做。
  - **方案 B**：同一设备内切换到学生端（若已登录角色）。

### 4.4 结构化反馈

- 表单字段建议：
  - 今晚任务是否完成（是/部分/否）
  - 最大困难点（单选 + 简短补充）
  - 是否需要老师联系（是/否）
- 提交后：**感谢态** + 回到收件箱或留在详情页显示「已反馈」。

---

## 5. 学生端：页面级流程与交互

### 5.1 今日任务 / 练习入口

- 列表或单页：仅显示当前练习（Hackathon 可只做一条）。
- 显示来源：哪张学习卡（弱化老师信息，突出题目）。

### 5.2 做题页

- 一屏一题或一屏多题（简化为多题即可）。
- 提交后：**即时反馈**（对错 + 一句解释）— 若时间允许；否则仅收集答案。

### 5.3 感受打卡

- 情绪选择 + 可选一句话。
- 提交成功页：**鼓励文案** + 返回结束。

---

## 6. 跨端一致性与命名

- **同一对象名称**：学习卡（Teacher Card / Parent Card）在三端统一叫「学习卡」。
- **家长端「行动区」**与老师生成结构一致，避免 AI 每次换版式。
- **状态词**：未读 / 已读 / 已反馈 / 已完成（家长任务）/ 已提交（学生）。

---

## 7. Hackathon 演示建议（帮助你们拍视频）

- 固定一条 **学科 + 年级 + 知识点** 的脚本数据；拍视频时可选用 **2.2** 的「一元二次方程」脚本。
- 演示顺序严格按 **2.1 主链路**（或照 **2.2** 的具体对白与操作）点击，不要跳端。
- 老师端重点展示：**预览 + 编辑 + 发送**；家长端重点展示：**行动区 + AI 解释 + 出题**；学生端重点展示：**提交 + 情绪**。
